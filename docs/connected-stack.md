# Connected Stack

This Linux surface is intentionally narrower than the full stack.

That is a feature, not a loss.

## What we preserve

We still preserve the relationship between:

- the Linux stack
- the broader agent and commons work
- the higher-level organizational and operator surfaces

## What we avoid

We avoid forcing every visitor to absorb the entire connected system on first load.

The landing page should stay about the Linux stack.

The architecture and FAQ pages can carry the connective tissue more quietly.

## Practical public rule

Connection should be present.

Connection should not become message blur.
